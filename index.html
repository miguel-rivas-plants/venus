DrLogic

This was the second project that i worked at Pixel Perfect Tree. When it was giving to me it was an empty frontend with the provided design and a defined .env.sample with a basic configuration of the ports, gitignored, ruby version and some initial gems like haml-rails and autoprefixer.

I install the missing gems for bootstrap, rail-sass and proposed to the lead developer a new way to defined metadata for every project. i wasn't sure how to do that in my second project in ruby on rails but i had an idea because i made something similar in Jade for my portfolio and in PHP for Cerveza Presidente. the specifications were a function which gets all the parameters that metadata will need:

  • Page details
  • Page title
  • Keys for SEO
  • Icons in differents resolutions
  • URL for Open Graph image (Facebook Preview)

We agree and started an investigation for that issue them began to deploy a function similar but the parameters were in the translation file (something that was new for me but really handy). After that the project was ready to begin.

There were a few things missing in the design that i made and them implemented:

  • Page 404
  • Facebook Preview
  • Loading
